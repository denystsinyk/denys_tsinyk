  <name>Task 1: Create GitHub Actions deploy workflow</name>
  <files>.github/workflows/deploy.yml</files>
  <action>
    Create `.github/workflows/deploy.yml` with the exact structure below. Do NOT use `peaceiris/actions-gh-pages` — use the official `actions/deploy-pages@v4` pipeline only.

    ```yaml
    name: Deploy to GitHub Pages

    on:
      push:
        branches: [main]
      workflow_dispatch:

    permissions:
      contents: read
      pages: write
      id-token: write

    concurrency:
      group: pages
      cancel-in-progress: true

    jobs:
      build:
        runs-on: ubuntu-latest
        steps:
          - uses: actions/checkout@v4

          - uses: actions/setup-node@v4
            with:
              node-version: '20'
              cache: 'npm'

          - run: npm ci

          - uses: actions/configure-pages@v5
            id: pages

          - name: Build
            run: npm run build
            env:
              VITE_BASE_PATH: ${{ steps.pages.outputs.base_path }}

          - uses: actions/upload-pages-artifact@v3
            with:
              path: dist

      deploy:
        needs: build
        runs-on: ubuntu-latest
        environment:
          name: github-pages
          url: ${{ steps.deployment.outputs.page_url }}
        steps:
          - id: deployment
            uses: actions/deploy-pages@v4
    ```

    Key correctness checks:
    - `permissions.pages: write` and `permissions.id-token: write` are REQUIRED — without them the deploy step fails with "Resource not accessible by integration"
    - `actions/configure-pages@v5` id must be `pages` so the output reference `steps.pages.outputs.base_path` resolves correctly
    - `VITE_BASE_PATH` env var is passed to the build step — this sets the Vite `base` to `/denys_tsinyk/` automatically in CI
    - `path: dist` in upload-pages-artifact must match Vite's `build.outDir` (default is `dist`)
    - `workflow_dispatch` trigger allows manual re-runs from the Actions tab
