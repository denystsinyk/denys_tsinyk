Create `scripts/` directory and `scripts/fetch-github-data.js`. This is a plain Node.js ESM script (no TypeScript, no npm packages — uses native fetch available in Node 18+).

The script must:

1. **Read env vars** — `GH_TOKEN` (PAT) and `GH_LOGIN` (username). If either is missing, exit 1 with a clear error message.

2. **Call GitHub GraphQL** with this query (combined pinned repos + contributions in one request):
```javascript
const query = `
  query($login: String!) {
    user(login: $login) {
      pinnedItems(first: 6, types: REPOSITORY) {
        nodes {
          ... on Repository {
            name
            description
            url
            stargazerCount
            primaryLanguage { name color }
          }
        }
      }
      contributionsCollection {
        contributionCalendar {
          totalContributions
          weeks {
            contributionDays { contributionCount date }
          }
        }
      }
    }
  }
`
```

3. **Compute top languages** by counting `primaryLanguage` occurrences across pinned repos. Sort descending by count. Use `primaryLanguage.color` from GraphQL directly — do NOT maintain a static color map (RESEARCH.md anti-pattern).

4. **Compute contribution streak** using the algorithm from RESEARCH.md Pattern 4:
   - Flatten all `contributionDays` from all weeks
   - Sort descending by date
   - Walk backwards from today counting consecutive days with `contributionCount > 0`
   - Break on first zero-contribution day
   - Note: year-boundary limitation is known and acceptable for v1

5. **Build the data.json payload** matching the SiteData shape from RESEARCH.md Pattern 1:
```javascript
const payload = {
  updated_at: new Date().toISOString(),
  steam_ok: false,         // Phase 3 will set this to true
  spotify_ok: false,       // Phase 4 will set this to true
  github: {
    pinned_repos: pinnedRepos,   // array of PinnedRepo objects
    stats: {
      topLanguages,              // { name, color, count }[]
      contributionStreak,        // computed integer
      totalContributionsThisYear // from GraphQL totalContributions
    }
  },
  steam: {
    top_games: [],
    currently_playing: null
  },
  spotify: {
    recent_tracks: [],
    currently_playing: null
  }
}
```

6. **Write to public/data.json** using `fs.writeFileSync` with `JSON.stringify(payload, null, 2)`.

7. **Error handling**: If GitHub GraphQL returns an error OR the HTTP status is non-2xx, log the error and exit 1 — this causes the pipeline to fail loudly (not silently). Do NOT overwrite `public/data.json` on error; keep the existing last-good-data.

Use `import` syntax at the top:
```javascript
import fs from 'node:fs'
import path from 'node:path'
import { fileURLToPath } from 'node:url'
```

Write `public/data.json` relative to the repo root using:
```javascript
const __dirname = path.dirname(fileURLToPath(import.meta.url))
const outputPath = path.join(__dirname, '..', 'public', 'data.json')
```

Add `"type": "module"` to `package.json` if not already present, OR run the script explicitly with `node --input-type=module` in the workflow. Check `package.json` — if it already specifies `"type": "module"` skip this. If not, keep the script as a `.mjs` extension to avoid breaking existing CJS assumptions.

IMPORTANT: GitHub username is read from `process.env.GH_LOGIN` — it is NEVER hardcoded in the script (user decision from CONTEXT.md).

  <name>Task 2: Create refresh-data.yml cron workflow</name>
  <files>.github/workflows/refresh-data.yml</files>
  <action>
Create `.github/workflows/refresh-data.yml` following RESEARCH.md Pattern 5 exactly:

```yaml
name: Refresh Data

on:
  schedule:
    - cron: '*/10 * * * *'   # every 10 minutes (GitHub minimum is 5 min)

permissions:
  contents: write             # required for git push back to main

jobs:
  refresh:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: '20'

      - name: Fetch GitHub data
        run: node scripts/fetch-github-data.js
        env:
          GH_TOKEN: ${{ secrets.GH_PAT }}
          GH_LOGIN: ${{ github.repository_owner }}

      - name: Commit updated data.json
        run: |
          git config user.name "github-actions[bot]"
          git config user.email "github-actions[bot]@users.noreply.github.com"
          git add public/data.json
          git diff --cached --quiet || git commit -m "chore: refresh data [skip ci]"
          git push
```

KEY implementation details:
- `permissions: contents: write` at workflow level (required since 2021 Actions hardening — without this, `git push` fails with 403)
- `GH_LOGIN: ${{ github.repository_owner }}` — uses built-in Actions context, no manual variable needed
- `GH_TOKEN: ${{ secrets.GH_PAT }}` — uses the PAT stored as `GH_PAT` repo secret (see user_setup)
- `git diff --cached --quiet || git commit` — only commits if data actually changed; prevents empty commits (RESEARCH.md Pitfall 4)
- `[skip ci]` in commit message — prevents `deploy.yml` from rebuilding the site on every 10-minute data refresh (RESEARCH.md anti-pattern avoidance)
- No `npm install` step needed — the script uses only Node built-ins

Do NOT add `workflow_dispatch` trigger — that is PIPE-04 which is deferred to v2 (REQUIREMENTS.md).
